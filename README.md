# 학습 관리 시스템(Learning Management System)
## 진행 방법
* 학습 관리 시스템의 수강신청 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 코드 리뷰 요청을 한다.
* 코드 리뷰 피드백에 대한 개선 작업을 하고 다시 PUSH한다.
* 모든 피드백을 완료하면 다음 단계를 도전하고 앞의 과정을 반복한다.

## 온라인 코드 리뷰 과정
* [텍스트와 이미지로 살펴보는 온라인 코드 리뷰 과정](https://github.com/next-step/nextstep-docs/tree/master/codereview)

## 개요
이 프로젝트는 이전 단계에서 구현한 도메인 모델(객체 구조)을 최대한 유지하면서 데이터베이스 테이블과 매핑하는 것을 목표로 합니다.  
성능보다는 도메인 로직의 구현에 초점을 맞추어, 여러 번의 DB 쿼리가 발생하더라도 객체 구조와 도메인 주도 설계(DDD)를 유지하는 데 중점을 둡니다.

## 구현할 기능 목록

- [x] **도메인 객체 및 테이블 매핑**
    - [x] 도메인 모델 구조를 그대로 유지하며 DB 테이블과 매핑
    - [x] 객체 간 의존성과 비즈니스 로직은 도메인 내에 구현
    - [x] Spring JDBC를 활용한 CRUD 기능 구현 (JdbcCourseRepository 참고)
    - [x] Repository 단위 테스트 (CourseRepositoryTest) 작성 및 검증


- [x] **데이터 초기화 및 스키마 관리**
    - [x] `src/main/resources/schema.sql` 파일에 테이블 생성 구문 정의 (예: course 테이블)
    - [x] `src/main/resources/data.sql` 파일을 활용한 샘플 데이터 초기화
    - [x] 데이터 초기화 정책 설정 (중복 데이터 삽입 방지 및 auto-increment 등 검토)

- [x] 강의 수강신청은 강의 상태가 모집중일 때만 가능하다.
- [x] 강의가 진행 중인 상태에서도 수강신청이 가능해야 한다.
- [x] 강의 진행 상태(준비중, 진행중, 종료)와 모집 상태(비모집중, 모집중)로 상태 값을 분리해야 한다.
- [x] 강의는 강의 커버 이미지 정보를 가진다.
- [x] 강사가 승인하지 않아도 수강 신청하는 모든 사람이 수강 가능하다.
- [x] 특정 강의는 수강생을 선발 할 수 있어야 한다.
- [x] 강사는 수강신청한 사람 중 선발된 인원에 대해서만 수강 승인이 가능해야 한다.
- [x] 강사는 수강신청한 사람 중 선발되지 않은 사람은 수강을 취소할 수 있어야 한다.

## 참고 사항
- 성능 최적화보다는 도메인 객체 내 로직 구현에 중점을 두고, 여러 번의 DB 쿼리 실행을 허용합니다.
- Payment 관련 결제 정보는 payments 모듈을 통해 별도로 관리합니다.
- 테스트 환경에서는 스키마 및 데이터 초기화 정책에 주의하여, 중복 데이터 삽입 등으로 인한 이슈가 없도록 설정합니다.
