# 학습 관리 시스템(Learning Management System)
## 진행 방법
* 학습 관리 시스템의 수강신청 요구사항을 파악한다.
* 요구사항에 대한 구현을 완료한 후 자신의 github 아이디에 해당하는 브랜치에 Pull Request(이하 PR)를 통해 코드 리뷰 요청을 한다.
* 코드 리뷰 피드백에 대한 개선 작업을 하고 다시 PUSH한다.
* 모든 피드백을 완료하면 다음 단계를 도전하고 앞의 과정을 반복한다.

## 온라인 코드 리뷰 과정
* [텍스트와 이미지로 살펴보는 온라인 코드 리뷰 과정](https://github.com/next-step/nextstep-docs/tree/master/codereview)


## 요구사항
### 질문 삭제하기
- 질문 데이터의 상태를 삭제 상태로 변경한다.
- 답변이 존재한다면 답변 또한 삭제 상태로 변경한다.
- 질문과 답변 삭제 이력에 대한 정보를 DeleteHistory를 활용해 남긴다.
#### 질문 삭제 가능 case
- 답변이 존재하지 않는다면
  - 로그인 사용자와 질문자가 같은 경우
- 답변이 존재한다면
  - 로그인 사용자와 질문자와 답변 글의 모든 답변자가 같은 경우
- 위 상황에 해당하는 경우에만 삭제가 가능하며, 그 외 경우는 모두 삭제할 수 없다.
#### 질문 삭제 불가능 case
- 공통 : 로그인 사용자와 질문자가 다른 경우
- 답변이 존재한다면
  - 로그인 사용자와 답변자가 다른 경우
  - 질문자와 답변자가 다른 경우
- 위 상황에 해당하는 경우 질문을 삭제할 수 없다.


## 모델링
### Answer
- `Answer`는 자신이 삭제 가능한 상태인지 아닌지 판단할 수 있다.

### Question
- `Question`은 질문을 삭제할 수 있다.
  - 이 때 `삭제`란 실제로 데이터를 삭제하는 것이 아니라, 데이터를 삭제 상태로 변경한다.
  - 질문이 삭제 가능한 경우 질문과 답변 삭제 이력에 대한 정보를 DeleteHistory를 활용해 남긴다.
  - 질문을 삭제할 수 없다면 예외를 발생시킨다.

### NsUser
- `NsUser`는 현재 글(질문, 답변)이 자신이 작성한 글인지 아닌지 판단할 수 있다.
