# 2단계 - 수강신청(도메인 모델)

## 기능 요구사항
- 과정(Course)은 기수 단위로 운영하며, 여러 개의 강의(Session)를 가질 수 있다.
- 강의는 시작일과 종료일을 가진다.
- 강의는 강의 커버 이미지 정보를 가진다.
  - 이미지 크기는 1MB 이하여야 한다.
  - 이미지 타입은 gif, jpg(jpeg 포함), png, svg만 허용한다.
  - 이미지의 width는 300픽셀, height는 200픽셀 이상이어야 하며, width와 height의 비율은 3:2여야 한다.
- 강의는 무료 강의와 유료 강의로 나뉜다.
  - 무료 강의는 최대 수강 인원 제한이 없다.
  - 유료 강의는 강의 최대 수강 인원을 초과할 수 없다.
  - 유료 강의는 수강생이 결제한 금액과 수강료가 일치할 때 수강 신청이 가능하다.
- 강의 상태는 준비중, 모집중, 종료 3가지 상태를 가진다.
- 강의 수강신청은 강의 상태가 모집중일 때만 가능하다.
- 유료 강의의 경우 결제는 이미 완료한 것으로 가정하고 이후 과정을 구현한다.
  - 결제를 완료한 결제 정보는 payments 모듈을 통해 관리되며, 결제 정보는 Payment 객체에 담겨 반한된다.

## 기능 구현 목록
- [x] Session을 생성한다
  - [x] 강의는 시작일과 종료일을 가진다
  - [x] 강의 커버 이미지 정보를 가진다
    - [x] 하단의 조건을 만족하지 않으면 예외가 발생한다
    - [x] 이미지 크기 제한은 1MB 이하이다
    - [x] 이미지 타입은 gif, jpg, jpeg, png, svg만 허용한다
    - [x] 이미지의 width는 300픽셀 이상, height는 200픽셀 이상이어야 한다
      - [x] width와 height의 비율은 3:2여야 한다
  - [x] 강의는 무료 또는 유료 강의이다
    - [x] 강의는 무료 또는 유료 강의 정책을 가진다
    - [x] 강의는 수강료를 가진다
      - [x] 수강료가 음수라면 예외가 발생한다
    - [x] 무료 강의는 수강 인원 제한이 없다
      - [x] 무료는 비용이나 수강 인원 관련 유효성 검사를 하지 않는다
    - [x] 유료 강의는 강의 최대 수강 인원을 초과할 수 없다
      - [x] 초과하면 예외가 발생한다
    - [x] 유료 강의는 수강 정원을 가진다
       - [x] 수강 정원이 음수일 때 예외가 발생한다
  - [x] 강의 수강신청은 강의 상태가 모집중일 때만 가능하다
    - [x] 수강신청 관련 정보를 가진다
      - [x] 중복이 발생한다면 예외가 발생한다
    - [x] 모집중 상태가 아니라면 예외가 발생한다
      - [x] 모집중 상태인지 확인한다

## 구현 방향 요약

- 도메인 전반에 영향을 주는 기능이 많아 한 번에 큰 구조를 만들기보다 값 객체와 enum처럼 의존성이 적은 요소부터 단계적으로 구현했다.
- 이미지, 정책, 수강신청 등 책임이 커질 수 있는 영역은 역할을 명확히 나누어 각 클래스가 하나의 책임에 집중하도록 설계했다.
- 무료/유료 강의의 수강 규칙을 SessionPolicy로 분리해 역할을 명확히 했다.
- 수강신청은 여러 규칙이 동시에 적용되는 행위이므로 Session을 Aggregate Root로 두고 Enrollment 생성을 통제했다.
- 구현이 안정된 이후에 패키지를 역할 기준으로 분리해 구조를 정리했다.
